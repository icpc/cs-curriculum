Week 2 - Floating Point Arithmetic

Some reading is in order especially for those of you new to competitive programming. Understanding things like representable range of values for various primitive data types and the limitations of precision due to floating point number representation in computers are both very important if you want to be successful.

A document that can really help you understand the issues around all things concerning floating point arithmetic in computers is:

What every computer scientist should know about floating point arithmetic. [pdf]
Read sections from the beginning through Relative Error and Ulps and Languages and Compilers through Optimizers

This will give you a complete description of floating point arithmetic and how it is handled by hardware, software, and compilers including the impact of things like substitutions (e.g. x/2.0 -> x * 0.5 vs x/10.0 -> x * 0.1) and expression simplification like x*y - x*z -> x*(y-z).

It may require a number of readings and writing some example code before it really starts to sink in.

There are 10 problems in this week's assignment. Before attempting any of the problems, complete the usual analysis of the problems:

Analysis

  • Understanding (What do they want?)
  • Recognition (Ahh! That's what they want and here's how I'll do it)
  • Ranking (trivial, easy, medium, hard, impossible)
  • Sizing (time / space complexity)
  • Time Estimate (how long to code up)
  • Ordering (how would you order these problems for solving)
